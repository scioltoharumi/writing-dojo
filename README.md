# writing-dojo

小説執筆力（特に演出力＝情報設計・視点距離・文体制御）を鍛えるための自己トレーニング用サイト。
敬愛する作家「無職転生 〜異世界行ったら本気だす〜」（作: 理不尽な孫の手、https://ncode.syosetu.com/n9669bk/ ）の実シーンを教材の土台にする。

要件定義・設計判断の経緯は本リポジトリではなく、制作ワークスペース側の以下に集約している（SSoT）:

```
00_admin/steering/20260712-writing-dojo/
├── requirements.md   要件定義本体
├── tasklist.md       実装タスクの進捗
└── decisions.md      主要決定事項とその理由
```

## 現状（プロトタイプ段階）

`data/questions.json` は**構成確認用のプレースホルダー**。無職転生からの実際の引用ではなく、ダミーテキストで6カテゴリの出題形式・回答例の見せ方を検証している段階。
サイト構成が固まったら、本格的な話数選定・引用抽出・回答例作成（要件定義7章）に進み、このJSONを本番データに差し替える。

## 構成

サーバー・DB・ビルドツールなし。素のHTML/CSS/JavaScript＋単一JSONファイル（`daily-research`と同じ構成方針）。

```
.
├── index.html   画面本体
├── app.js       出題ロジック・描画・localStorage保存
├── style.css    モバイルファーストのスタイル
└── data/
    └── questions.json   問題バンク（カテゴリ×複数問）
```

## デプロイ時の必須ルール

**JS/CSS/データを変更したら、必ず `version.json` の `version` を更新してからpushする。**

`index.html` は起動役（ブートストラップ）で、毎回 `version.json` をキャッシュ迂回で取得し、その番号付きURLで `app.js` / `style.css` を読み込む。これにより、ブラウザやCDNに古いHTMLが残っていても、開くたびに最新のJS/CSSが読み込まれる。`version.json` の更新を忘れると、変更したファイルが利用者に届かない。

現在読み込まれているバージョンは、サイトのフッターに `version YYYY-MM-DD.N` として表示される。

## 動作確認

`fetch` を使うため `file://` 直接開きでは動かない。簡易サーバー経由で確認する。

```bash
python -m http.server 8000
# → http://localhost:8000/
```

## 6カテゴリ

1. 分析的写経（フランクリン式再構築）
2. シーンのリバースエンジニアリング
3. 同一シーン多重リライト
4. 制約ドリル
5. 音読とレンズ別推敲
6. カメラ距離の意識化

各カテゴリは日付（JST基準）とカテゴリ名から決定的に1問を選ぶ擬似ランダムで日替わり出題する。回答は端末のlocalStorageにのみ保存される（サーバー同期なし）。
